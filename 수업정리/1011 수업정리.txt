소멸자 특징
객체가 소멸될 때 자동으로 호출되어 자동 반환 작업을 수행

소멸자는 클래스에 소속된 멤버함수이다.
1. 소멸자 이름은 클래스 이름으로 한다.
2. 소멸자는 생성자와 구분하기 위해 함수명 앞에~ 기호를 붙인다.
3. 소멸자는 자료형을 지정하지 않는다. 반환값이 없다.
4. 소멸자는는 객체가 소멸할 때 컴파일러에 의해 자동으로 호출되며, 명시적으로 호출하지 않는다.
5. 소멸자는 public으로 선언되어야 한다.
6. 소멸자는 매개변수를 지정할 수 없고, 함수 오버로딩도 불가능하다.
프로그램에서 소멸자 생략 : 소멸자도 자동으로 생성. 함수의 몸체가 비어있는 아무 일도 하지 않는 빈껍데기 소멸자이다. -> 소멸자를 디폴트 소멸자

접근 제어
외부에서 특정한 멤버 변수나 멤버 함수에 접근하는것을 제어하는 것.

private나 public 등의 접근 지정자를 멤버 변수나 멤버 함수 앞에 붙여서 접근을 제한한다.
private가 붙으면 전용 멤버로 되어서 클래스 안에서만 사용 가능하다.
public을 붙이면 모든 사람들이 볼 수 있는 공용 멤버가 되어 다른 모든 클래스들이 사용 가능하다.

객체와 함수
객체와 함수와의 관계
1. 객체가 함수의 매개변수로 전달되는 경우
함수의 인수는 기본적으로 값에 의하여 매개 변수로 전달된다. 이것은 인수가 객체일 때도 마찬가지이다.
객체를 함수로 전달하는 경우에는 객체가 복사된다.
함수에서 매개 변수를 어떻게 변경하건 원본 객체에는 영향을 주지 않는다.
하지만 객체의 크기가 커지면 객체를 복사하는 것도 상당히 시간이 걸린다.
call by value와 같음

2. 객체의 참조자가 함수의 매개변수로 전달되는 경우
참조자란 변수에 하나의 이름을 더 주는 것이다. &기호를 이용하여 정의한다.
참조자를 통하여 값을 변경하면 원본 변수를 변경하는 것이나 마찬가지이다. 객체에 대한 참조자도 마찬가지의 역할을 한다.
일반적으로 객체를 값으로 전달하는 것보다 객체의 참조자를 전달하는 편이 더 효울적이다. 객체의 크기가 큰 경우에 객체를 복사하는 시간이 많이 걸리기 때문이다.

3. 함수가 객체를 반환하는 경우
함수가 객체를 반환할 때에도 객체의 내용이 복사될 뿐 원본이 전달되지 않는다.
